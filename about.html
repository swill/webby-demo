About The Project

A Browser-Native
Web Editor

Webby makes owning and updating your website as natural as editing a document — no CMS, no terminal, no deployment pipeline to wrangle.

Capabilities

What You Can Do
with Webby

Webby is a complete publishing workflow inside a single JavaScript file. Here's what becomes possible the moment you load it with your secrets file:

1

Edit Text Directly On-Page

Any element marked data-editable becomes a live contenteditable field. Click it, type, done.

2

Replace Images Instantly

Click any data-editable-image to open a file picker. The image uploads to your GitHub repo automatically.

3

Generate Sections with AI

Hover between sections, click + Add Section, describe what you want in plain English — Webby calls Claude to generate fully structured HTML.

4

Adjust Your Theme Live

The Theme panel lets you tweak colours, fonts, and spacing in real-time and see changes instantly across the entire page.

5

Publish Without a Terminal

Click Publish. Webby serialises your page, strips all editor code and credentials, and pushes directly to GitHub via the Contents API. GitHub Actions deploys it in ~60 seconds.

Under The Hood

Two Modes.
One Clean Design.

Webby's architecture is elegantly simple: the same HTML file behaves completely differently depending on whether secrets.js is present.

Edit Mode (Local)

When secrets.js Is Present

Opening the site locally with a secrets.js alongside it activates Webby's full editor. The dark toolbar appears at the top of the page, editable zones highlight on hover, and AI tools come online.

  • ✦   Inline text contenteditable activated
  • ✦   Image click-to-replace enabled
  • ✦   AI section insertion available
  • ✦   Theme editor live
  • ✦   Publish button pushes to GitHub
Public Mode (Deployed)

The Deployed Site

When Webby publishes, it serialises the page and strips all editor-related scripts, credentials, toolbar elements, and data attributes used solely for editing. The result is lean, clean HTML — no overhead, no exposure.

  • ✦   Zero editor code in production
  • ✦   No API keys in deployed HTML
  • ✦   No JavaScript dependencies on the live site
  • ✦   Fast, static, and secure
  • ✦   Works on any static host

The Content Database Is The HTML File

There is no database, no API, no CMS backend. The HTML file is the content store. When you edit and publish, Webby writes the updated HTML directly to your GitHub repository. Your local folder is never a git repository — Webby handles everything through the GitHub Contents API.
