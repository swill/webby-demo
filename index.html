Free & open source

Edit your website by clicking on it.

Webby turns any static HTML file into an editable, publishable website — right in your browser. No code, no CMS, no developer needed. Works with any static site and is hosted for free on GitHub Pages.

Get Started View on GitHub
Simple by design

Three steps. That's it.

1
Open

You open index.html in your browser. Because secrets.js is present, Webby activates and the editor appears at the top of the page — no installation, no setup.

2
Edit

Click any text to change it, click any link to update its URL, or click any image to replace it. Hover between sections and click + Add Section to describe what you want — the AI generates it, styled to match your site.

3
Publish

Hit the Publish button. Webby commits your changes directly to GitHub and your live site updates automatically — usually within about a minute.

Starting from scratch?

Start a new site in minutes

Webby doesn't just edit existing sites — it also helps you create new ones from scratch.

You use a prompt with Claude.ai (no account needed beyond the free tier) to generate a complete, styled index.html from a description of your business. The prompt instructs Claude to analyse inspiration images for colours and typography, write realistic placeholder content, and wire up all the data attributes Webby needs — so the editor works immediately on the generated file.

Example prompt 
You are generating the initial downloadable index.html for a professional website.

I am attaching [N] inspiration images. Analyze them for:
- Dominant and accent colors → translate to a CSS variable palette
- Typography mood (serif/sans-serif, weight, spacing)
- Overall aesthetic tone (minimalist, warm, clinical, natural, bold, etc.)

Use those signals to define a cohesive CSS variable system and base styles.

---

BUSINESS DETAILS:
Name:         [Full name or practice name]
Profession:   [e.g. Osteopath, Naturopath, Coach, Photographer]
Tagline:      [One sentence — what they do and for whom]
Location:     [City, Country]
Contact:      [Email and/or phone to appear on site]
Social:       [Instagram, LinkedIn, etc. — or leave blank]

---

TONE & AESTHETIC:
[Describe the feeling the site should evoke. Examples:
- "Calm, trustworthy, professional. Like a high-end wellness clinic."
- "Warm and approachable. Earthy tones, clean layout, not too corporate."
- "Minimal and modern. Lots of whitespace, strong typography."
Use your own words — be as specific or loose as you like.]

---

SECTIONS TO INCLUDE:
[List each section and what it should contain. Examples:

1. Hero — Full-width header with name, tagline, and a call-to-action button ("Book a Session")
2. About — Photo + 2–3 paragraphs about background, philosophy, approach
3. Services — 3 service cards: Osteopathy, Sports Therapy, Online Consultations
4. Testimonials — 3 short quotes from clients
5. FAQ — 4 common questions with answers
6. Contact — Email, phone, location, contact form (static mailto: is fine)
7. Footer — Name, copyright, social links

Adjust this list to match the actual site.]

---

TECHNICAL REQUIREMENTS:
Generate a single index.html file with the following:

1. DOCTYPE html5, semantic HTML, mobile-first responsive layout
2. No external CSS frameworks — all styles inline in a <style> block
3. A CSS custom property system using the variables defined from the images:
   --color-primary, --color-secondary, --color-accent,
   --color-bg, --color-bg-alt, --color-text, --color-text-muted,
   --font-heading, --font-body,
   --space-xs through --space-xl,
   --container-width, --radius, --shadow
4. Google Fonts import for the chosen typefaces (2 max)
5. Each section must have:
   - data-zone="{slug}" on the <section> element
   - data-zone-label="{Human Label}" on the <section> element
   - data-editable on every user-editable text element (headings, paragraphs, spans)
   - data-editable-image on every <img> element
6. A single <nav> with smooth-scroll links to each section
7. Responsive mobile navigation (hamburger toggle using only CSS or minimal inline JS)
   Mobile nav requirements — these are mandatory, not optional:
   - The expanded mobile menu must be fully visible on screen and never clipped
   - Use position: fixed or position: absolute (not static/relative) for the expanded menu so it
     escapes any parent overflow constraints
   - Set a z-index high enough (e.g. 9000) that the menu appears above all page content
   - Never rely on a parent container with overflow: hidden to contain the menu
   - The menu must be scrollable (overflow-y: auto) if it could exceed viewport height
   - Test the toggle logic: the menu must open AND close correctly on repeated hamburger clicks
8. No JavaScript frameworks — vanilla JS only, and only where necessary
9. Include this script tag in the <head>, immediately after the <style> block:
   <script src="./secrets.js"></script>
   <script src="https://[YOUR_CDN]/webby.js"></script>
   (These will be stripped on export — they enable edit mode locally)
10. All placeholder text should be realistic and relevant to the profession
11. Placeholder images should use: <img src="./assets/placeholder.jpg" data-editable-image alt="..." />
12. The overall visual result should be polished, professional, and production-ready

---

OUTPUT:
Return ONLY the complete index.html file contents.
No explanation, no markdown fences, no preamble.
Start your response with: <!DOCTYPE html>

This is the exact prompt format used to generate the site you're looking at right now. Attach 2–5 photos that capture the look and feel you're going for — screenshots of sites you like, colour palette images, photos from your business — and Claude will translate them into a cohesive visual design.

Let's do this

Ready? Here's how to set it up.

Everything you need is free — GitHub (free), Claude.ai (free tier), and a text editor to create one small file. If you can copy and paste, you can do this.

  1. 1
    Create a GitHub account and repository

    Sign up at github.com if you don't have an account. Create a new public repository for your site (e.g. your-name-website). Enable GitHub Pages under Settings → Pages → Deploy from branch → main. Your site will be live at your-username.github.io/your-repo.

  2. 2
    Generate your index.html with Claude

    Open Claude.ai, paste in the Bootstrap Prompt above, attach a few inspiration images, and fill in your business details. Claude will return a complete, styled HTML file ready for Webby. Save it as index.html in a folder on your computer.

  3. 3
    Create your secrets.js file

    In the same folder as index.html, create a plain text file called secrets.js. Add your Gemini API key (free from Google AI Studio — no billing required), your GitHub personal access token, and your repository name. This file stays on your computer and is never published to GitHub.

  4. 4
    Open index.html in your browser

    Double-click the file. For the best experience use Chrome, Edge, or Safari — they support auto-saving your edits directly back to disk. Firefox works too, but you'll need to use the Export button to save your changes each session. The Webby editor will appear automatically at the top of the page.

  5. 5
    Link your site folder

    A banner will appear below the toolbar on first open. Click Select Folder and choose the folder containing your index.html. Once linked, every edit auto-saves to your local file within a couple of seconds — so your changes are never lost if you close the tab or restart your browser.

  6. 6
    Hit Publish

    When you're happy with your edits, click the Publish button in the Webby toolbar. Webby will commit your changes to GitHub and your site will be live within about a minute. Every publish is a git commit, so your full history is always there.

  7. 7
    Add a custom domain

    You can add a custom domain by updating your Github Pages configuration. Get more details by reviewing the docs.

What does secrets.js look like?
window.SITE_SECRETS = {
  geminiKey:   "AIza...",          // Google AI Studio — free at aistudio.google.com
  githubToken: "ghp_...",          // GitHub fine-grained PAT (contents: read + write)
  repo:        "your-username/your-repo",
  branch:      "main"
};

Got questions?

Common questions

Do I need to know how to code?
Not at all. You edit text by clicking on it, replace images by clicking them, and add new sections by describing what you want in plain English. The only technical step is creating the secrets.js file — which is just copying a short template and filling in three values.
Is this really free?
Webby itself is free and open source (MIT license). GitHub Pages hosting is free. Claude.ai has a free tier sufficient for generating your initial site. The Gemini API key used for AI section generation inside Webby is also free — Google AI Studio provides a generous free tier with no billing required.
What happens to my credentials?
Your API keys and tokens live only in secrets.js on your own computer. They are never committed to GitHub, never sent anywhere except directly to the Google and GitHub APIs, and never stored by Webby. You are always in control.
Can I use this for an existing site?
Yes, as long as your site is a single HTML file hosted on GitHub Pages. You'll need to add the data-zone, data-editable, and data-editable-image attributes to your HTML, and add the two script tags. The CLAUDE.md file in the Webby repository documents all the attributes in detail.
What if I want to customize the code or self-host webby.js?
Webby is open source at github.com/swill/webby. Fork the repository, make your changes, and update the script tag in your index.html to point to your own hosted copy. The README has instructions for releasing your own version.
